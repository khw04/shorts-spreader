\# 쇼츠 살포기 (Shorts Spreader) — Implementation Spec



\## Summary



Chrome 확장 프로그램 기반 실시간 쇼츠 공유 시스템.

사용자가 YouTube Shorts 페이지에서 "살포하기" 버튼을 누르면, 같은 확장 프로그램을 설치한 다른 사용자들의 브라우저에서 현재 페이지의 미디어 태그(`<img>`, `<video>`) 중 랜덤 1개가 해당 쇼츠 iframe으로 교체된다.



\---



\## Tech Stack



\- \*\*Fullstack:\*\* Next.js (App Router)

\- \*\*WebSocket:\*\* `ws` 라이브러리, custom server (`server.js`)로 Next.js + WebSocket을 같은 포트에서 운영

\- \*\*상태 관리:\*\* 인메모리 (DB 없음, 해커톤 규모)

\- \*\*배포:\*\* Railway (커스텀 서버 지원, WebSocket 상시 연결 가능)

\- \*\*도메인:\*\* w00.kr 연결

\- \*\*Chrome Extension:\*\* Manifest V3, 순수 JS

\## Custom Server (server.js)



Next.js의 request handler와 WebSocket 서버를 같은 HTTP 서버에 붙인다.



```javascript

// 구조 예시

const { createServer } = require('http');

const { parse } = require('url');

const next = require('next');

const { WebSocketServer } = require('ws');



const app = next({ dev: process.env.NODE\_ENV !== 'production' });

const handle = app.getRequestHandler();



app.prepare().then(() => {

&#x20; const server = createServer((req, res) => handle(req, res, parse(req.url, true)));

&#x20; const wss = new WebSocketServer({ server });



&#x20; wss.on('connection', (ws) => { /\* 메시지 핸들링 \*/ });



&#x20; server.listen(process.env.PORT || 3000);

});

```

\## WebSocket Protocol



\### Client → Server



\*\*살포 요청:\*\*

```json

{

&#x20; "type": "spread",

&#x20; "payload": {

&#x20;   "shortsUrl": "https://www.youtube.com/shorts/VIDEO\_ID",

&#x20;   "shortsTitle": "영상 제목",

&#x20;   "spreaderName": "닉네임"

&#x20; }

}

```



\*\*피격 확인 (미디어 교체 성공 시):\*\*

```json

{

&#x20; "type": "hit\_confirm",

&#x20; "payload": {

&#x20;   "spreadId": "uuid",

&#x20;   "victimName": "닉네임",

&#x20;   "replacedTagType": "img | video",

&#x20;   "pageUrl": "피격된 사이트 도메인"

&#x20; }

}

```



\*\*대시보드 등록 (대시보드 페이지 접속 시):\*\*

```json

{

&#x20; "type": "register\_dashboard"

}

```



\### Server → Client



\*\*피격 알림 (살포자 제외 전원, 대시보드 제외):\*\*

```json

{

&#x20; "type": "hit",

&#x20; "payload": {

&#x20;   "shortsId": "VIDEO\_ID",

&#x20;   "shortsTitle": "영상 제목",

&#x20;   "spreaderName": "닉네임",

&#x20;   "spreadId": "uuid"

&#x20; }

}

```



\*\*통계 업데이트 (전원):\*\*

```json

{

&#x20; "type": "stats\_update",

&#x20; "payload": {

&#x20;   "activeUsers": 14,

&#x20;   "totalSpreads": 123,

&#x20;   "totalHits": 456

&#x20; }

}

```



\*\*살포 이벤트 (대시보드 전용):\*\*

```json

{

&#x20; "type": "spread\_event",

&#x20; "payload": {

&#x20;   "spreaderName": "닉네임",

&#x20;   "shortsTitle": "영상 제목",

&#x20;   "victimCount": 5,

&#x20;   "spreadId": "uuid",

&#x20;   "timestamp": "ISO string"

&#x20; }

}

```



\*\*피격 이벤트 (대시보드 전용):\*\*

```json

{

&#x20; "type": "hit\_event",

&#x20; "payload": {

&#x20;   "spreadId": "uuid",

&#x20;   "victimName": "닉네임",

&#x20;   "replacedTagType": "img",

&#x20;   "siteDomain": "naver.com",

&#x20;   "timestamp": "ISO string"

&#x20; }

}

```



\### Server-side Message Handling



```

on "spread":

&#x20; 1. spreadLog에 기록

&#x20; 2. stats.totalSpreads++, spreadsPerUser\[name]++

&#x20; 3. 살포자 제외 모든 extension 클라이언트에 "hit" 브로드캐스트

&#x20; 4. 대시보드 클라이언트에 "spread\_event" 전송

&#x20; 5. 전원에게 "stats\_update" 전송



on "hit\_confirm":

&#x20; 1. stats.totalHits++, hitsPerUser\[victimName]++, hitSites\[domain]++

&#x20; 2. 대시보드 클라이언트에 "hit\_event" 전송

&#x20; 3. 전원에게 "stats\_update" 전송



on "register\_dashboard":

&#x20; 1. 해당 클라이언트를 isDashboard: true로 마킹

&#x20; 2. "hit" 브로드캐스트 대상에서 제외



on connection:

&#x20; 1. clients에 추가, peakActiveUsers 갱신

&#x20; 2. stats\_update 브로드캐스트



on close:

&#x20; 1. clients에서 제거

&#x20; 2. stats\_update 브로드캐스트

```

\### background.js



역할: WebSocket 연결 유지, content.js ↔ 서버 메시지 브릿지.



```

1\. 확장 프로그램 활성화 시 서버에 WebSocket 연결

2\. 닉네임은 chrome.storage.local에서 로드 (없으면 랜덤 생성 후 저장)

3\. content.js → background.js: chrome.runtime.sendMessage

4\. background.js → content.js: chrome.tabs.sendMessage

5\. 연결 끊김 시 exponential backoff 재연결 (1s → 2s → 4s → max 30s)

```



서버 URL은 상수로 정의:

```javascript

const WS\_URL = "wss://your-domain.com"; // 배포 시 변경

```



\### content.js



\#### 살포 기능 (youtube.com/shorts/\* 에서만)



```

1\. URL 패턴 체크: /shorts/(\[a-zA-Z0-9\_-]+)/

2\. 매치 시 "🔥 살포하기" FAB 버튼을 페이지에 주입

3\. YouTube는 SPA이므로 URL 변경 감지 필요:

&#x20;  - MutationObserver로 URL 변경 감지

&#x20;  - 쇼츠 페이지 진입 시 버튼 생성, 이탈 시 제거

4\. 버튼 클릭 시:

&#x20;  - URL에서 VIDEO\_ID 파싱

&#x20;  - 제목은 DOM에서 파싱 시도 (실패 시 빈 문자열)

&#x20;  - chrome.runtime.sendMessage({ type: "spread", ... })

&#x20;  - 버튼 텍스트 "살포 완료! 💨"로 변경 (1.5초 후 복귀)

```



\#### 피격 기능 (모든 페이지, youtube.com 제외)



```

chrome.runtime.onMessage 리스너로 "hit" 수신 시:



1\. 교체 대상 수집:

&#x20;  - document.querySelectorAll('img:not(\[data-salpo]), video:not(\[data-salpo])')

&#x20;  - 필터: width >= 80 \&\& height >= 80 \&\& 화면에 보이는 것

2\. 대상이 있으면:

&#x20;  - 랜덤 1개 선택

&#x20;  - 원래 크기 저장 (offsetWidth, offsetHeight)

&#x20;  - YouTube embed iframe으로 교체:

&#x20;    src = https://www.youtube.com/embed/{VIDEO\_ID}?autoplay=1\&mute=0\&loop=1\&playlist={VIDEO\_ID}

&#x20;  - iframe에 data-salpo="true" 부여

&#x20;  - 교체 애니메이션 (opacity + scale 트랜지션)

&#x20;  - 화면 상단 토스트: "\[닉네임]이(가) 쇼츠를 살포했습니다! 💥" (3초 후 사라짐)

&#x20;  - hit\_confirm 메시지 전송

3\. 대상이 없으면:

&#x20;  - 화면 중앙 오버레이 모달로 쇼츠 표시

&#x20;  - 닫기 버튼은 3초 후 활성화

```

\### 대시보드 (/dashboard)



WebSocket으로 서버에 연결 (register\_dashboard), 실시간 데이터 표시.



\*\*상단: 통계 카드 4개\*\*

\- 🟢 접속자 수 (activeUsers)

\- 🔥 총 살포 횟수 (totalSpreads)

\- 💥 총 피격 횟수 (totalHits)

\- 🖼️→📺 교체된 미디어 (= totalHits)

\- 숫자 변경 시 카운트업 애니메이션



\*\*중앙 좌측: 실시간 살포 피드\*\*

\- spread\_event, hit\_event를 시간순 표시

\- 형식: `\[HH:MM:SS] 🔥 닉네임 → 쇼츠 제목 / 💥 피해자 피격! (사이트)`

\- 새 이벤트 slide-in 애니메이션

\- 최대 50건 유지



\*\*중앙 우측: 네트워크 시각화 (Canvas 또는 SVG)\*\*

\- 접속자를 원형 배치한 노드 그래프

\- 살포 시 살포자→피격자 방향 파티클 애니메이션

\- 노드 크기: 살포 횟수 비례

\- 노드 색상: 피격 횟수 비례 (많을수록 빨간색)



\*\*하단: 리더보드\*\*

\- 살포왕 TOP 5 / 피격왕 TOP 5



\*\*디자인 토큰:\*\*

\- 배경: #0a0a0f

\- 카드 배경: rgba(255,255,255,0.05), backdrop-filter: blur(10px)

\- 포인트 컬러: 빨강 #FF3B30, 오렌지 #FF9500, 네온그린 #30FF50

\- 폰트: 제목은 모노스페이스, 본문은 산세리프

\## Technical Notes



\### YouTube Shorts URL → Embed

```

파싱: url.match(/\\/shorts\\/(\[a-zA-Z0-9\_-]+)/)?.\[1]

Embed: https://www.youtube.com/embed/{VIDEO\_ID}?autoplay=1\&mute=0\&loop=1\&playlist={VIDEO\_ID}

```



\### Manifest V3 Service Worker 제약

\- 비활성 상태에서 깨어나야 하므로 WebSocket 끊길 수 있음

\- `chrome.alarms`으로 주기적 연결 체크, 또는 Offscreen Document API 활용

\- 닉네임/설정은 `chrome.storage.local`에 저장



\### Content Script SPA 대응

\- YouTube는 SPA이므로 URL 변경을 감지해야 함

\- `MutationObserver` 또는 `chrome.webNavigation.onHistoryStateUpdated` 사용

\- 쇼츠 페이지 진입/이탈 시 살포 버튼 생성/제거



\### 미디어 교체 범위

\- MVP: `<img>`, `<video>` 태그만 대상

\- `<picture>` 내부, `background-image`, Shadow DOM 내부는 무시

\- youtube.com에서는 피격 기능 비활성화 (자기 페이지 보호)



\### Railway 배포

\- `server.js`를 엔트리포인트로 지정 (Procfile 또는 package.json scripts)

\- `package.json`의 `start` 스크립트: `node server.js`

\- PORT 환경변수를 Railway가 자동 주입

\- WebSocket은 `wss://` (Railway가 TLS 처리)
